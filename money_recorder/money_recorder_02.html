Theme

まず、Themeを書いていきましょう。Android Studioを立ち上げて少々待つとこのような画面になると思います。

右上のほうの赤丸が打ってある、三本線と四角のアイコン(Split)を押してください。

するとこのように、画面が出てきます。
これはMainActivity.kt内にある@Previewアノテーションがついたコンポーザブル関数がプレビューされたものが表示されます。
Kotlinには少し慣れが必要だと思うので、とりあえずプレビューしたいものにはつけるものなんだと、覚えておいてください。

次にui>themeにある、Color.kt Theme.kt Type.ktを改変していきましょう。主にコピペです。
まず下の、リンクの Material Theme Builder に飛んでください。
ダウンロードが面倒な方は、下にコピペを張るのでそれを使ってください。

Material Theme Builder - https://material-foundation.github.io/material-theme-builder/

このサイトはアンドロイドベースの UI のベース的なのをコピペさせてくれるサイトです。好みの画像を選択して、好みの色使いを選んで右下の Pick your fonts → で進んでください。
次はフォントの選択画面ですが、今回は何も選択しません。右下の Export theme → を選択して、次に進みましょう。
また次の色の薄い右下の Export を押すと次の画面になると思います。

そして、Jetpack Compose (Theme.kt) を選択して、ダウンロードされた zip ファイルを展開してください。

これをメモ帳などで開いて、Android Studio のほうに書き換えてください。
ここで注意点なのですが、一番上のpackage_com.sakura.money_recorder.ui.themeを変えるとエラーが出るので、それ以外をコピペしましょう。

また、もう一つの注意点で、コピペするとTheme.ktのfun MoneyRecorderThemeがfun AppThemeに変えられてしまうので戻しましょう。

また、Color.ktにおいて HighContrast や MediumContrast などの使わない色まで書いてあるのでいらないと思うのであれば消してください。残すのは一番上のと、ダークモードのやつです。

@Composable
fun MoneyRecorderTheme(
    darkTheme: Boolean = isSystemInDarkTheme(),
    // Dynamic color is available on Android 12+
    dynamicColor: Boolean = true,
    content: @Composable() () -> Unit
) {
    val colorScheme = when {
        dynamicColor && Build.VERSION.SDK_INT >= Build.VERSION_CODES.S -> {
            val context = LocalContext.current
            if (darkTheme) dynamicDarkColorScheme(context) else dynamicLightColorScheme(context)
        }

        darkTheme -> darkScheme
        else -> lightScheme
    }

    MaterialTheme(
        colorScheme = colorScheme,
        typography = AppTypography,
        content = content
    )
}

今書いたものはスクリーンのそのうち必要になる関数の一式です。まず HomeScreen() は実機で使われることになるコンポーザブル関数です。 コンポーザブル関数とは @Composable アノテーションがついている関数で、画面に表示される機能を持つ者には大体つきます。もしこれが赤線ついてたりすると 一番上の import_androidx.compose.runtime.Composable がないからだと思われます。これを書かなくても自動入力してくれたりもします。 その中にある HomeScreenBody() はホーム画面を書く本体の関数です。その下の @Preview アノテーションがついた PreviewHomeScreen() はBodyの関数をプレビューするための関数です。 この三つがなくてもいいんじゃないかと思われるかもしれませんが、のちに ViewModel を導入する際にそれはプレビューできなくいろいろ困るので、今後を考えた一式となっています。

まずは Scaffld() の今回使う大事な部分だけ書き、少し説明します。もしいろいろな文字が赤文字になっている場合はまた、importが足りていません。 その赤線にマウスを合わせると修正の提案が出てくると思うので、Alt + Enterなどで自動修正ができると思うので import を追加してください。
引数の topBar bottomBar floatingActionButton に中かっこ{}を代入している形になっています。{}はPythonでいうラムダ式のようなもので新しく関数を作って代入するのではなく{}の中にもう関数を書くという方法です。 これもKotlinのちょっとした癖で、慣れれば結構わかりやすくて、私は好きです。
またScaffoldの{}部分にinnerPadding ->というものが書いてあると思いますが、これはScaffoldの戻り値をinnerPaddingと名付けて使うといったことを書いています。 これを書かない場合名前は初期値のitになります。それをわかりやすくinnerPaddingとなずけて使用しているというわけです。 もし、itでもinnerPaddingでも使わないと赤文字になりますので、忘れないようにしましょう。 これも理解が難しく、私もはじめは全然わからなかったのですが、今後同じようなことを何回も書くのでそのうちになんとなくでもわかっていくと思います。
innerPadding自体の戻り値の意味なのですが、Column()関数は中のコンポーザブル関数を縦に並べて配置していくという関数で、それの背景のサイズなどを自動でやってくれる的なものです。 昔はSurface()などが使われていたそうですが、最近はScaffold()がこのinnerPaddingの機能によりよく使われています。

@OptIn(ExperimentalMaterial3Api::class) //バージョンの条件指定のため必要
@Composable
fun AppTopBar( //引数を書く欄
    titleText: String, //上に表示される名前
    canBack: Boolean = false, //戻るボタンの有無設定 初期値はfalse
    onBackButtonClicked: () -> Unit = {} //関数の引数、初期値は空の関数{}
) {
    CenterAlignedTopAppBar(
        colors = TopAppBarDefaults.topAppBarColors(
            containerColor = MaterialTheme.colorScheme.primaryContainer,
            titleContentColor = MaterialTheme.colorScheme.primary,
        ),
        title = { Text(titleText) },
        navigationIcon = {
            if (canBack)
                IconButton(
                    onClick = onBackButtonClicked
                ) {
                    Icon(
                        imageVector = Icons.AutoMirrored.Default.ArrowBack,
                        contentDescription = null
                    )
                }

またcom.___.money_recorderの下にFunctions.ktをFileで作りましょう。 そして、次の関数をコピペしてください。

Functions.kt
import android.icu.text.SimpleDateFormat
import java.text.NumberFormat
import java.util.Date
import java.util.Locale

fun getNowMillis(): Long {
    return System.currentTimeMillis()
}

fun convertMillisToDate(millis: Long): String {
    val formatter = SimpleDateFormat("yyyy.MM.dd", Locale.getDefault())
    return formatter.format(Date(millis))
}

fun convertMillisToYearMonth(millis: Long): String {
    val formatter = SimpleDateFormat("yyyy年MM月", Locale.getDefault())
    return formatter.format(Date(millis))
}

fun formatYen(num: Int): String {
    val numberFormat = NumberFormat.getNumberInstance(Locale.getDefault())
    return "${numberFormat.format(num)} 円"
}

面倒くさいので今後使うであろう関数四つを追加します。
getNowMillis()は現在時刻をLong型で取得する関数です。お金の記録とき、初期値としてこの関数を使用します。
convertMillisToDate()はLong型をyyyy.mm.ddの形式にフォーマットして変換する関数です。
convertMillisToYearMonth()は先ほどとは違う形式でフォーマットする関数です。moneyRecordGroupにまとめるときにname=に代入して使います。
formatYen()は1000という数字などを1,000 円とフォーマットする関数です。

ここで少しKotlinの関数について書きます。上のような関数の書き方は手続き型のような書き方です。プログラミングの知識のある方ならお分かりではあると思いますが、

関数の書き方例
fun Function(/* 引数 */) : /* 戻り値 */ {
    /* 処理 */
    return ---
}

このような構文になっています。しかし、Kotlinにはオブジェクト指向型のようなクラス内蔵のような関数も書くことができます。

オブジェクト指向型関数の書き方例
fun ClassA.Function(/* 引数 */) : ClassA = ClassA(
    time = getNowMillis()
)

このようにあるクラスインスタンスに対し特定の処理をした新しいクラスにするような関数も書くことができます。そのうち出てくると思います。
